# Memento

Memento is a dementia care companion focused on two roles only: caregiver and patient. Caregivers manage patient records, routines, medications, memory notes, and emergency contacts. Patients sign in with a unique patient ID and see the same synchronized record through a simplified portal.

## Core Experience

- Caregiver sign-up and sign-in with email and password
- Patient sign-in with a unique caregiver-generated patient ID
- SQLite-backed storage for caregiver accounts, patients, tasks, medications, contacts, summaries, and assistant history
- Caregiver dashboard with multi-patient support, guided patient intake, editable patient memory/profile data, medication management, emergency contacts, and notifications
- Patient dashboard with synced routine tasks, assistant chat, medication tracking, emergency contacts, SOS alerts, and end-of-day summary logging
- Privacy Policy and Accessibility pages linked from the footer

## Stack

- React 18
- TypeScript
- Vite
- Node.js HTTP API
- SQLite via Node's built-in `node:sqlite`
- Tailwind CSS
- shadcn/ui
- React Hook Form + Zod

## Local Setup

```bash
npm install
npm run dev
```

`npm run dev` starts both the local SQLite API server on `http://127.0.0.1:3001` and the Vite client on `http://127.0.0.1:8080`.

## Scripts

- `npm run dev` starts the API server and frontend together
- `npm run dev:server` starts only the SQLite API server
- `npm run dev:client` starts only the Vite frontend
- `npm run build` creates a production build
- `npm run start` starts the Node server for the built app
- `npm run lint` runs ESLint
- `npm run test` runs the Vitest suite

## Authentication Model

- Caregivers create accounts with full name, email, and password.
- Caregivers can create patient records after signing in.
- Each patient receives a unique `MEM-XXXXXX` style patient ID generated by the backend.
- Patients do not use passwords and cannot self-register from the auth screen.

## Data Storage

- The app stores its local database in `data/memento.sqlite`.
- Passwords are stored as hashed values.
- The frontend talks to the backend through `/api` routes proxied by Vite during development.
