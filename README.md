# Memento

Memento is a full-stack dementia care companion built for two connected roles only: caregiver and patient. It helps caregivers manage patient records, routines, medications, emergency contacts, and memory-support notes, while giving patients a calmer, simplified portal backed by the same synchronized record.

The product is designed around a practical care workflow:
- Caregivers create and manage patient profiles.
- Each patient receives a unique login ID generated by the backend.
- Patients sign in with that ID only and view the information their caregiver has prepared for them.
- Both roles interact with the same shared care record, so updates stay in sync across portals.

## Why Memento Exists

Dementia care often depends on repetition, reassurance, structure, and fast access to the right people. Memento brings those needs into one place:

- Routines and reminders help patients stay grounded through the day.
- Medication schedules reduce missed or duplicated doses.
- Emergency contacts stay accessible when fast action matters.
- Care notes and memory prompts give the assistant more useful context.
- Caregivers can monitor updates without overwhelming the patient experience.

## Core Features

### Caregiver Portal

- Register with email and password
- Add and manage multiple patients
- Generate a unique patient login ID for each patient
- Set support level and dementia classification
- Maintain a patient memory profile for the assistant
- Add and remove medications
- Add and remove routine tasks
- Add emergency contacts
- Review patient summaries and notifications
- Use the care note assistant to prepare calmer, clearer guidance

### Patient Portal

- Sign in with a caregiver-issued patient ID
- View routines and medications from the shared record
- Talk to the Memento Assistant
- Mark medications and tasks as completed
- Add personal tasks to the day
- Save an end-of-day summary
- Access emergency contacts and trigger SOS alerts

## Local Development

### Prerequisites

- Node.js 22 or later
- npm 10 or later recommended

### Install

```bash
npm install
```

### Configure AI

Create a local environment file:

```bash
cp .env.example .env.local
```

Then add your OpenAI key to `.env.local`:

```env
OPENAI_API_KEY=your_api_key_here
OPENAI_MODEL=gpt-5.4-mini
```

If `OPENAI_API_KEY` is missing, the app still runs, but live AI assistant responses will be disabled.

### Run The App

```bash
npm run dev
```

This starts:

- The API server at `http://127.0.0.1:3001`
- The Vite frontend at `http://127.0.0.1:8080`

### Production Build

```bash
npm run build
npm run start
```

`npm run start` serves the built frontend from `dist/` and runs the backend from the same Node server.

## Available Scripts

- `npm run dev`: Start the frontend and backend together
- `npm run dev:client`: Start only the Vite frontend
- `npm run dev:server`: Start only the API server in dev mode
- `npm run build`: Create a production frontend build
- `npm run start`: Start the production server
- `npm run lint`: Run ESLint
- `npm run test`: Run the Vitest suite
- `npm run test:watch`: Run tests in watch mode
- `npm run smoke:api`: Run a lightweight backend smoke test against a running API server

## How To Use Memento

### Caregiver Flow

1. Open the app and create a caregiver account.
2. Enter the caregiver dashboard.
3. Add a patient and complete the guided intake questions.
4. Save the generated patient login ID.
5. Add medications, routine tasks, contacts, and memory notes.
6. Share the patient login ID with the patient or their trusted support person.

### Patient Flow

1. Open the app and choose patient access.
2. Sign in using the caregiver-issued patient ID.
3. Review today’s routine and medication schedule.
4. Use the assistant for reminders or reassurance.
5. Add personal tasks for the day if needed.
6. Save an end-of-day summary for the caregiver to review.

## Data And Security Notes

- Caregiver passwords are hashed before storage.
- Patient access is ID-based and does not require a password.
- Local development data is stored in `data/memento.sqlite`.
- The app includes Privacy Policy and Accessibility pages in the public site.

## GitHub Actions

This repository includes GitHub Actions pipelines for validation and delivery:

- `ci.yml`
  - installs dependencies
  - runs lint
  - runs unit tests
  - builds the app
  - boots the API and runs a real smoke test against the auth/patient flow

- `delivery.yml`
  - runs on pushes to `main`
  - builds the production app
  - packages a release-ready artifact containing the built frontend, server, and runtime files

The delivery workflow is set up for continuous delivery of a deployable artifact. It does not auto-deploy to a hosting provider because this project currently has no production infrastructure configuration checked into the repository.
